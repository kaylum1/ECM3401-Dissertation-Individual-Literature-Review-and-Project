Tracker Detection Scan - Understanding Web Tracking

This page explains tracker detection scans, what it means when a scan does not detect anything, how users can be exposed, and the scientific process behind how these scans work.

What is Web Tracking?

In a short answer, web tracking is when information is collected on how people browse the internet, websites, advertisers, and other third-party services use this data for analytics, personalised ads, and content customisation.

Common tracking techniques include some of the following:

  • Cookies: These are tiny files placed on your device that record browsing behaviour and settings.
  • Fingerprinting: Gathering distinctive browser and device details to single out users.
  • Behavioural Logging: Capturing actions such as clicks, scrolling, and time spent on a page.
  • Tracking Pixels: Invisible images that relay user activity back to a server.

What Happens When a Scan Doesn't Detect Any Trackers?

If a tracker-detection scan reports zero trackers, it could mean:

  • The site avoids cookies, fingerprinting, and other tracking scripts.
  • A strict Content Security Policy (CSP) will blocks un-authorised scripts from loading.
  • Sophisticated or obfuscated trackers slipped past the scan.
  • The site deliberately anonymises data and honours user privacy.

Note: Clever tracking code can cloak itself, so a clean report is not a guarantee.

How Users are Left Exposed to Trackers

Tracking can lead to serious privacy concerns, including:

  • Loss of Privacy: Websites can track user behaviour across multiple domains.
  • Targeted Advertising: Companies build profiles based on browsing habits.
  • Data Leaks: If trackers collect personally identifiable information (PII), it may be exposed in breaches.
  • Price changes: Some websites adjust prices dynamically based on tracking data.
  • Reduced Online Anonymity: Browser fingerprinting allows websites to track users even if cookies are disabled.

How to Protect Yourself from Web Tracking

Users can reduce tracking exposure by following these best practices:

  • Use Privacy-Focused Browsers: Firefox, Brave, and DuckDuckGo prioritise privacy.
  • Install Tracker-Blocking Extensions: Use tools like Privacy Badger or uBlock Origin.
  • Block Third-Party Cookies: Disable third-party cookies in browser settings.
  • Use a VPN: VPNs can mask IP addresses, reducing location-based tracking.
  • Enable “Do Not Track” (DNT): Some websites respect this browser setting.
  • Disable JavaScript for Untrusted Sites: Many trackers rely on JavaScript execution.

The Science Behind Tracker Detection Scanning

Tracker detection scanners analyse web pages using several techniques:

  • Network Traffic Analysis: The scanner intercepts outgoing network requests to detect third-party tracking domains.
  • Cookie Inspection: It checks for cookies set by non-essential third-party services.
  • JavaScript Analysis: The scanner examines loaded scripts for known tracking behaviours.
  • Fingerprinting Detection: Identifies browser properties collected for tracking purposes.
  • Behavioural Simulation: Simulates user interactions to detect hidden tracking mechanisms.

Why Websites Should Reduce Trackers

Based on some different laws, websites should aim to limiting trackers benefits both site owners and visitors, and helps satisfy regulations such as:

  • General Data Protection Regulation (GDPR): Demands explicit consent before tracking.
  • California Consumer Privacy Act (CCPA): Gives users greater control over their data.
  • Privacy and Electronic Communications Directive (PECD): Governs cookie usage in the EU.
